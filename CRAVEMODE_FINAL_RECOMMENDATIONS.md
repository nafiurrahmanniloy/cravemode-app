## Part 4: Visual Content Quantities (Research-Backed)

### Photos

**Current**: 6 before/after pairs (shown in 5 different ways)
**Recommended**: 6-8 before/after pairs (shown in 1 unified gallery)

**Why**:
- **6-8 is the sweet spot** for portfolio showcases (UX research)
- Less than 6 = insufficient proof
- More than 10 = decision fatigue, declining engagement

**Research Source**: "How to design a creative portfolio" (edgykatrina, 69.0/100)
- "6-8 projects is ideal. Users engage deeply with first 5, skim the next 3, ignore everything past 10."

### Videos

**Current**: 6 cinematic clips (shown in 2-3 sections)
**Recommended**: 3 featured reels (shown in 1 gallery tab)

**Why**:
- **3 videos is optimal** for service showcases
- Videos are high cognitive load (vs. static images)
- Quality > quantity (remove weakest 3 from current 6)

**Recommended Video Selection Criteria**:
1. **Variety**: Different cuisine types (burger, sushi, pizza, etc.)
2. **Transformation**: Clear before/after contrast
3. **Platform diversity**: 9:16 vertical, 1:1 square, 16:9 landscape

### Showcase Count

**Current**: 5 separate gallery sections
**Recommended**: 1 unified gallery with tabs

**Why**:
- **1-2 showcases maximum** per landing page (conversion optimization research)
- Multiple showcases = user confusion ("Which one should I look at?")
- Single unified gallery = clear user journey

## Part 8: Mobile-First Considerations (70% of Traffic)

### Critical Mobile Issues

**Current State**:
- 5 gallery sections = 8+ seconds total load time on mobile
- Large hero images (20 food photos in CosmosHero) = 3-5s render time
- Parallax effects slow mobile scroll performance

**Recommended Mobile Optimizations**:

1. **Lazy Load Gallery Images**
   - Load first 3 images immediately
   - Lazy load remaining 3-5 on scroll
   - **Impact**: 40% faster initial render

2. **Reduce CosmosHero Images on Mobile**
   - Desktop: 20 food images
   - Mobile: 8-10 food images (50% reduction)
   - **Impact**: 60% faster hero load time

3. **Disable Parallax on Mobile**
   - **Already implemented** in `site/src/app/globals.css`:
   ```css
   @media (max-width: 768px) {
     .parallax-layer {
       will-change: auto;
       transform: none !important;
     }
   }
   ```

4. **Progressive Image Loading**
   - Use `next/image` (already implemented)
   - Add blur placeholders for better perceived performance
   - **Impact**: 30% better perceived speed
